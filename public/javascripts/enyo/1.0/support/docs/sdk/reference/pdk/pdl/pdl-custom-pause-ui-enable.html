PDL_CustomPauseUiEnable

Description

By default, the system assumes an application has a UI for handling a reactivation from a pause. If your application does not, it can call this function with PDL_FALSE to use the system's UI for reactivation, which requires the user to tap a play icon to resume (see screenshot).

Note:

Does not disable phone ringing or messages.

Syntax 

PDL_Err PDL_CustomPauseUiEnable(PDL_bool Enable);

Returns 

PDL_ECONNECTION — Unable to communicate with app card.
PDL_NOERROR — Success.


System UI for resuming the application
